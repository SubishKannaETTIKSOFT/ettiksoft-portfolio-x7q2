Detailed walkthrough · No prior experience needed

From a zip file to a working roadmap, in five minutes.

This guide assumes you have never opened a terminal before. Every step is spelled out for Windows, Mac, and Linux. If you get stuck, the troubleshooting section at the end covers the most common problems.

TL;DR if you're already comfortable: Extract the zip, cd into the folder, run python3 -m http.server 8000, open http://localhost:8000 in your browser. Done. Skip to Step 8 for using the app.
01

Save the zip file somewhere you'll remember

You should already have a file called roadmap-builder.zip. Move it somewhere easy to find — your Desktop is the simplest choice.

TIP Put it on your Desktop. Every step below assumes that's where it is. If you put it somewhere else, replace "Desktop" with that location in the commands.
02

Extract the zip file

A zip file is a compressed folder. You need to extract it before you can use what's inside.

Windows
  1. Find roadmap-builder.zip on your Desktop.
  2. Right-click the file.
  3. Click Extract All… in the menu that appears.
  4. A window opens. Click Extract (the default location is fine).
  5. You'll now see a new folder called roadmap-builder on your Desktop. Open it. You should see four files: index.html, editor.html, output.html, README.md.
Mac
  1. Find roadmap-builder.zip on your Desktop.
  2. Double-click the file.
  3. A new folder called roadmap-builder appears next to the zip — Mac extracts it automatically.
  4. Open the folder. You should see four files: index.html, editor.html, output.html, README.md.
Linux
  1. Right-click roadmap-builder.zip on your Desktop.
  2. Click Extract Here (or Open With Archive Manager then Extract).
  3. You'll see a new folder called roadmap-builder.
03

Open a terminal

A terminal (also called command prompt or shell) is a window where you type commands instead of clicking. We'll only use two simple commands.

Windows

You have two options. Either works. PowerShell is recommended.

Option A — PowerShell (recommended)

  1. Press the Windows key on your keyboard (the one with the Windows logo).
  2. Type PowerShell.
  3. Click Windows PowerShell in the search results.
  4. A blue window opens. That's your terminal.

Option B — Command Prompt

  1. Press the Windows key.
  2. Type cmd.
  3. Click Command Prompt.
  4. A black window opens. That's your terminal.
Mac
  1. Press Cmd + Space together. The Spotlight search bar opens.
  2. Type Terminal.
  3. Press Enter.
  4. A window opens with a prompt. That's your terminal.
Linux
  1. Press Ctrl + Alt + T together (works on most distributions).
  2. Or open your applications menu and search for Terminal.
WHAT IT LOOKS LIKE Whichever terminal you open, you'll see a window with text and a blinking cursor waiting for you to type. The line you'll type on is called the prompt. It usually ends with a >, $, or % sign.
04

Check if Python is installed

We need Python to run a tiny local server (60 seconds of work — it doesn't change anything on your computer permanently). Most Mac and Linux systems already have it. Windows usually does not, but installation is one click.

Type this command and press Enter

$ python3 --version

You'll get one of two responses:

GOOD RESPONSE Something like Python 3.11.4 (any 3.x version is fine). Skip to Step 5.
NEED TO INSTALL Something like 'python3' is not recognized or command not found. Install Python first using the instructions below for your OS, then come back to this step.

Installing Python (only if needed)

Windows
  1. Open https://www.python.org/downloads/ in your browser.
  2. Click the big yellow Download Python 3.x.x button.
  3. Run the downloaded installer.
  4. VERY IMPORTANT: on the first screen, check the box that says "Add python.exe to PATH" at the bottom. If you skip this, the command won't work.
  5. Click Install Now.
  6. Wait for it to finish. Click Close.
  7. Close your terminal window completely (the X in the top corner).
  8. Open a new terminal (Step 3 again) and run python3 --version. It should now report a version. If still not found, also try python --version (without the 3).
Mac

Mac comes with Python pre-installed. If python3 --version doesn't work, install via Homebrew:

$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# installs Homebrew (the Mac package installer)
$ brew install python3
Linux

Almost every Linux distribution ships with Python 3. If yours doesn't:

$ sudo apt install python3 # Ubuntu / Debian
$ sudo dnf install python3 # Fedora / RHEL
DON'T WANT TO INSTALL PYTHON? Alternatives: if you have Node.js, use npx serve . instead of the Python command in Step 6. If you have PHP, use php -S localhost:8000. If none of these — install Python; it's the easiest.
05

Move into the roadmap-builder folder

The terminal is currently "looking at" some default folder (usually your home folder). We need to tell it to look at the roadmap-builder folder you extracted in Step 2. We do this with the cd command (it stands for "change directory").

Type this and press Enter

Windows
PS C:\Users\You> cd Desktop\roadmap-builder

After Enter, your prompt should change to show you're now inside the folder:

PS C:\Users\You\Desktop\roadmap-builder> _
Mac
$ cd ~/Desktop/roadmap-builder

The ~ is shorthand for "your home folder." After Enter, no error means it worked.

Linux
$ cd ~/Desktop/roadmap-builder

Confirm you're in the right place

Type ls (Mac/Linux) or dir (Windows) and press Enter. You should see the four files listed:

$ ls
README.md editor.html index.html output.html
SHORTCUT On Mac and Linux, you can drag the roadmap-builder folder from Finder/Files onto your terminal window — it will paste the full path automatically. Then just type cd (with a trailing space) before it and press Enter.

On Windows in PowerShell: navigate to the folder in File Explorer, click in the address bar (or press Alt+D), type powershell, and press Enter — opens a terminal already inside that folder.
06

Start the local server

Now we run a tiny "web server" on your own computer. It only serves the four files in this folder, only to you, only while the terminal stays open. Nothing leaves your machine.

Type this and press Enter

$ python3 -m http.server 8000

You should see something like:

Serving HTTP on :: port 8000 (http://[::]:8000/) ...
SUCCESS That message means the server is running. Leave this terminal window open. If you close it, the server stops and the app won't load.
FIRST TIME ON WINDOWS Windows may ask permission for Python to access the network — click Allow. This is just so your browser can talk to the server on the same machine. Nothing goes to the internet.
IF IT SAYS "ADDRESS ALREADY IN USE" Port 8000 is being used by something else. Use a different port: python3 -m http.server 8123 (or any number between 1024 and 65000). Remember the number for Step 7.
07

Open the app in your browser

Open Chrome, Firefox, Safari, or Edge — any modern browser works. In the address bar at the top, type the URL below and press Enter.

http://localhost:8000

(If you used a different port in Step 6, replace 8000 with that number.)

What you should see

The home page of Roadmap Builder loads. It looks something like this:

localhost:8000
Roadmap Builder · Home
Build a roadmap your team actually uses.
Single-page roadmaps for product teams
+ Create new roadmap
YOU'RE IN If you see this, the app is working. Now we use it.
PAGE WON'T LOAD? Check that the terminal from Step 6 is still showing the "Serving HTTP" message. If you closed it accidentally, just repeat Step 6.
08

Create your first roadmap

Click the + Create new roadmap button on the home page. The page changes to the editor — you'll spend most of your time here.

The editor is divided into six sections. None of them are required — fill in what you want, leave the rest blank. Empty sections won't appear on the output page.

  • Identity — company name, tagline, date, version
  • Thesis — the headline statement of your strategy
  • KPIs — the key numbers anchoring your strategy
  • Strategic bets — the 2–4 outcomes everything else serves
  • Products — one card per product / workstream
  • 30-day actions — the list of things due this month
EVERYTHING AUTO-SAVES You don't need to click "save." A small indicator at the top right shows Saving… as you type, then All changes saved. You can close the tab or shut down — your work is safe in your browser's local storage.
09

Fill in the editor

Walk through it top to bottom. Below is what each section is for.

Roadmap name (top of the page)

Click the title at the top — it says Untitled roadmap by default. Type the name you want (e.g., "Q3 Product Strategy"). This is what appears on the home page list.

Section 01 — Identity

Top-bar metadata for the roadmap. Company name, optional tagline, the date you want shown, and a version number. All optional, but the company name is recommended.

Section 02 — Thesis

The single statement that summarizes your strategy. Four fields:

  • Eyebrow — a small uppercase line above the headline (e.g., "12-month strategy"). Optional.
  • Headline — the one sentence your team can repeat. Punchy, verb-led.
  • Accent word — type one word from your headline; it gets highlighted in italic + brand color in the output.
  • Lede — a 2–3 sentence elaboration. The supporting detail.

Section 03 — KPIs

The numbers that anchor your strategy (e.g., 5 products, 1 paid eval). Click + Add KPI to add a row. Each KPI has three fields:

  • Number — the big number (e.g., "5")
  • Subscript — small text next to it (e.g., "paid eval"). Optional.
  • Label — what the number means (e.g., "Customers in pipeline")

Keep to 4–6 KPIs for the cleanest output.

Section 04 — Strategic bets

The 2–4 outcomes everything else in the company is in service of. Each bet has:

  • Verb — one word (CONVERT, SHIP, DECIDE, BUILD, etc.)
  • Headline — the bet stated as a complete sentence
  • Why — 1–2 sentences on why this bet matters

Section 05 — Products

One card per product or major workstream. Each product has:

  • Name
  • Stage (e.g., "Pre-GA", "Beta", "Concept")
  • RAG — click one of three colored buttons. Green = on track, Amber = caution, Red = at risk.
  • Capacity — what's working on it (e.g., "2 devs · 60% PO")
  • Next milestone — the immediate goal
  • Why now — one sentence on relevance

Section 06 — 30-day actions

The list of things due in the next month. Each row has date (free text — write it however you want), what needs to happen, who owns it, and a priority (P0 = critical / P1 = important / P2 = nice).

REMOVING ITEMS Every row in a dynamic section has a small × in the top-right corner. Click it to remove that row.
NAVIGATION Click ← Home at the top-left to go back to the list. Click View output → at the top-right to see what your roadmap looks like rendered.
10

View the rendered roadmap

From the editor, click View output → in the top-right. From the home page, click the View button next to a roadmap.

This is the visual page that summarizes everything you typed — the "deliverable" you'd share with someone. Empty sections are hidden automatically, so the layout adapts to what you've filled in.

At the top there's a small toolbar with three actions:

  • ← Home — back to the list
  • ✎ Edit — back to the editor for this roadmap
  • ⎙ Print — opens your browser's print dialog (you can save as PDF from there)
  • ↓ Download HTML — see Step 11
PRINTING The page is print-friendly — the toolbar and background texture are removed automatically when you print or save as PDF. Use your browser's "Save as PDF" option in the print dialog.
11

Download as a standalone HTML file

Click the ↓ Download HTML button on the output page. Your browser downloads a single HTML file (named after your roadmap, e.g., Q3_Product_Strategy.html) into your default Downloads folder.

This downloaded file is completely self-contained. You can:

  • Email it as an attachment
  • Open it on any other computer by double-clicking — no app, no internet (except for fonts), no install
  • Embed it in Notion / Confluence / SharePoint
  • Host it on a webserver for a permanent link
  • Open it offline on a flight
WHAT'S DIFFERENT IN THE DOWNLOADED FILE It has no toolbar (no Edit / Print / Download buttons) and no JavaScript. Just the rendered roadmap, fully styled. View-only.
REGENERATE IF YOU CHANGE THINGS The downloaded file is a snapshot. If you go back and edit the roadmap, click ↓ Download HTML again to get an updated copy.
12

When you're done — stop the server

When you're finished using the app, return to the terminal window from Step 6.

Press Ctrl + C

(That's the Control key and the letter C, pressed together. On Mac, it's still Ctrl, not Cmd.)

The server stops. You can close the terminal.

YOUR DATA IS SAFE Stopping the server doesn't affect your saved roadmaps. They live in your browser's local storage and stay there until you delete them or clear browser data.
13

Using it again later

Once everything is set up, getting back to work takes 20 seconds.

  1. Open a terminal (Step 3)
  2. cd ~/Desktop/roadmap-builder (Mac/Linux) or cd Desktop\roadmap-builder (Windows)
  3. python3 -m http.server 8000
  4. Open http://localhost:8000 in your browser

Your previously created roadmaps will be there. Same browser, same machine = same data.

DIFFERENT BROWSER OR DIFFERENT COMPUTER LocalStorage is per-browser-per-device. If you switch from Chrome to Firefox, or move to a different laptop, your saved roadmaps won't appear. To move them, see "Backup & restore" below.
↓

Backup & restore

Your roadmaps are stored in your browser. Browsers can clear data unexpectedly (especially in private browsing). Back up regularly if your roadmaps matter.

Quick backup

Open the home page of the app. Press F12 to open browser DevTools (or right-click → Inspect). Click the Console tab. Paste this and press Enter:

copy(localStorage.getItem('roadmap-builder.roadmaps'))

Your data is now copied to clipboard. Paste it into a text file and save somewhere safe.

Restore

On the new browser / computer (after Steps 1–7 of this guide), open the home page, F12 → Console, and run:

localStorage.setItem('roadmap-builder.roadmaps', PASTE_BACKUP_HERE)

Replace PASTE_BACKUP_HERE with your backed-up text, wrapped in single quotes if it isn't already. Refresh the page.

Easier alternative

For each important roadmap, click ↓ Download HTML and save the file. The downloaded HTML files are backups — they don't lose any visible content even if your browser data goes away. The only thing you lose is the ability to re-edit (you'd have to recreate it in the editor).

Troubleshooting

The most common problems and how to fix them.

The terminal says 'python3' is not recognized or command not found

Python isn't installed, or it's installed but not in your "PATH." Go back to Step 4 and install it (Windows users: make sure to check the "Add to PATH" box during installation). After installing, close and reopen the terminal.

On some systems the command is just python, not python3. Try python --version. If that works, use python -m http.server 8000 in Step 6.

The terminal says Address already in use or OSError: [Errno 48]

Port 8000 is being used by another program. Pick a different port: python3 -m http.server 8123 (any number 1024–65000 works). Then visit http://localhost:8123 in your browser instead.

The browser says "This site can't be reached" or just spins

Check that the terminal from Step 6 is still showing the "Serving HTTP" line. If it has stopped, restart it. If you accidentally closed the terminal, the server is gone — open a new terminal and repeat Steps 5–6.

Also make sure the URL is http://localhost — not https://.

I created a roadmap, refreshed, and it's gone

Most likely you opened a private/incognito window — those don't persist localStorage. Use a normal browser window instead.

Also check you're using the same browser you created it in. Roadmaps in Chrome don't appear in Firefox.

The fonts look wrong / generic

Google Fonts is blocked by your network or your browser is offline. The app still works, just with system fonts instead. To fix: get internet access, or download the fonts locally and update the <link> tag in each HTML file to reference the local copies.

Download HTML button doesn't do anything

Some browsers block downloads from local pages. Try a different browser (Chrome / Edge / Firefox handle this well). Or right-click the page → Save As, then trim the file manually (advanced).

I want to start completely fresh — delete everything

F12 → Console, paste: localStorage.removeItem('roadmap-builder.roadmaps') and press Enter. Refresh the home page. All roadmaps gone.

I want to change the colors / fonts

Open output.html in a text editor (Notepad, TextEdit, VS Code, anything). Find the :root { block at the top of the <style> tag. Edit any of the color values (the hex codes like #1F3F4F). Save. Refresh the page. The change applies everywhere.

Something else is broken

Try: (1) refresh the page, (2) restart the server (Ctrl+C in terminal, then run the server command again), (3) try a different browser.
